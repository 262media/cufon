Limitations

Browser support

The following browers are supported:

  • Safari 2+ (note: need to verify 2.0)
  • Firefox 1.5+ (note: need to verify 1.5)
  • Opera 9+
  • Internet Explorer 6+

Additionally any standards-compliant browser with support for <canvas> should work just fine.

There's a delay before the text is replaced!

The most common causes are the following:

  1. You have a slow-loading external <script> before the closing </body> tag. This will prevent the DOMContentLoaded Event (and friends) from firing in time. Another option is to manually call Cufon.DOM.ready() without any arguments just before the other files. This will tell Cufon that the DOM is ready to be modified.
  2. You are executing other JavaScript before calling Cufon.replace(). Depending on what you're doing this might cause a delay. If possible, you should always call Cufon.replace() first and then do the rest of whatever you want to do.

My font has blocky edges! How come?

There are limits to how accurately you can draw with <canvas>, which cuf√≥n uses. Your font just happens to have curves that cannot be drawn so well. Changing the scaling options rarely helps, but using a different font size might give slightly better results in some cases.

Ligatures & Digraphs

For now ligatures and digraphs are unsupported due to performance reasons. Support for them might be added in a future release but even then it would only be made available as an option that would be disabled by default.
